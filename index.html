배경 및 목표

기존 검색의 한계를 극복하고 사용자 의도를 이해하는 검색으로

❌ AS-IS

정확 매칭 중심

  • 지역/골프장명/키워드 기반 정확 매칭
  • "경기", "솔트베이", "노캐디"처럼 100% 맞는 단어만 검색됨
  • "경기 라운딩", "주말 라운딩" 같은 문장형 → 0건 빈번
  • "그린 빠른 골프장" 같은 의도형 검색 불가
  • 탐색 실패로 인한 이탈 증가

✓ TO-BE

자연어 의도 이해

  • 사용자는 자연어로 자유롭게 검색
  • 기존 검색(정확매칭) 먼저 실행 → 속도 보장
  • 0건일 때만 컨텍스트 에이전트 실행
  • 캐시 우선 + 필요 시 툴(리뷰DB/예약DB/GPT) 호출
  • 결과 UI는 기존 골프장 리스트 100% 유지

03

툴별 커버 범위

3가지 핵심 도구와 각각의 역할

🗄️
Tool 1: 골프 예약 DB

최종 결과 생성/검증 담당
모든 노출 결과는 반드시 Tool1 기준(골프 예약 API 조회)으로만 생성

  • 날짜/시간: 오늘/내일/주말/특정일, 새벽/오전/오후/야간
  • 지역/거리: 서울/경기/강원/충청/경상/전라, 근교→반경
  • 가격: n만원 이하, 가성비(상한/정렬 치환)
  • 인원/조건: 2/3/4인, 조인 가능
  • 옵션: 노캐디/셀프체크인/환급·할인/임박특가
  • 정렬: 추천/가격/거리/평점
⭐
Tool 2: 통합 리뷰 DB

체감/평판 속성 기반 후보 생성
카카오 리뷰 + 네이버 크롤링으로 예약DB에 없는 속성 커버

  • 그린 속도/컨디션 (빠름/느림/모래 많음)
  • 코스 관리/난이도/밀림 정도
  • 페어웨이 넓음/좁음, 블라인드홀, 벙커
  • 시설/클럽하우스 청결/고급스러움
  • 식당/음식 품질, 가성비
  • 캐디/직원 서비스, 접근성 체감
  • 대회 개최 이력 (KLPGA 등)
🌐
Tool 3: GPT (웹서치)

지식형/추론형/실시간 정보 보완
Tool2로 커버 불가한 지식형, 최신 정보, 외부 랭킹 담당

  • "명문 골프장" 같은 추상/명성 질의
  • "접대용으로 유명한 골프장"
  • Tool2 데이터 부족 시 "대회 개최 골프장"
  • 코스명/별칭/오타 교정 및 후보 생성
  • 최근 리모델링/신규 오픈 골프장
  • 골프 다이제스트/잡지 선정 베스트 코스
  • 프로 골퍼/연예인 추천 골프장
  • 링크스/파크/밸리 스타일 코스 분류
  • ※ 비용 발생! Tool2 우선 사용 원칙

📦 캐시 적재 대상

캐시 유형 내용 TTL (권장)
NLU 캐시 정규화된 intent/slots 1~7일
후보 캐시 (핵심) 질의키 → 후보 코스ID TopK + 랭킹점수 + 사용한 슬롯 1~7일
리뷰 속성 캐시 (aspect, 지역 등 조건) → 코스ID TopK + 점수/근거 카운트 1~7일
GPT 후보/매핑 캐시 지식 질의 → 코스명 리스트/ID 매핑 결과 7~30일
웹서치 결과 캐시 최신 정보 질의 → 골프장 리스트 + 출처 3~14일
트렌드 요약 캐시 (선택) 최근 개선/악화 요약 7일
⚠️

캐시 금지 항목

티타임 잔여/가격/잔여팀은 변동성이 커서 장기 캐시 금지
최종 노출 직전에는 항상 Tool1(예약DB)로 재검증 필수

📋 MVP 범위 (확정)

  • ✓ 기존 검색(정확매칭) 유지
  • ✓ 0건 시 컨텍스트 에이전트 실행

    명문/접대, 최근 리모델링, 잡지 선정, 코스 스타일

"정확매칭은 즉시 기존 검색으로,

최종은 예약DB 조회 결과만 기존 리스트 UI에 노출한다."

"캐시는 GPT 결과만이 아니라

툴 결과 중 재사용 가치 있는 것들을 적재한다."
